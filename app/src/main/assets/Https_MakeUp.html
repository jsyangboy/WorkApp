相关知识

HTTP

http超文本传输，是明文传输，即直接传输没有经过加密的信息， 这个的信息很容易被拦截，篡改，对个人的敏感信息非常危险，HTTP 使用的TCP协议，在七层网络结构中属于应用层

秘密学

对称加密:

对称加密又叫私钥加密，即信息发送方和接受方使用同一个密钥去加密和解密 ，对称加密的特点是算法是公开的，加密跟解密速度快，合适于对数据量进行加密
常见的对称加密算法有
DES，3DES，TDEA，Blowfish，RC5和IDEA。

<> 其加密过程如下：
明文+加密算法+密钥 = 密文 解密过程如下：
密文+解密算法+密钥 = 明文

非对称加密：

非对称加密也是公钥加密，非对称加密于对称加密相比，其安全性更好 ，对称加密的通信双方使用相同的密钥，如果一方的密钥遭到泄露，那个真个通信就会被破解 ，而非对称加密使用一对密钥，即公钥跟私钥，且二者是成对出现的，私钥被自己保存 ，不能对外泄露，公钥指的是公共的密钥，任何人都可以获得该密钥，使用公钥或者私钥中的 任何一个进行加密，用另外一个进行解密 非对称加密的缺点是加密跟解密花费时间长，速度慢，只适合少量的数据进行加密
常见的非对称加密使用的算法有：
RSA，Elgmal，Rabin，D-H，ECC（椭圆曲线加密）

被公钥加密过的密文只能私钥解密：
加密：明文+加密算法+公钥=秘文
解密：密文+解密算法+私钥=明文

被私钥加密过的密文只能公钥解密：
加密：明文+加密算法+私钥 = 密文
解密：密文+解密算法+公钥 = 明文

HTTPS

HTTPS = HTTP + SSL/TLS ，即在在信息传输的过程对中信息增加了 加密，HTTPS为了兼顾安全跟效率，加密同时使用了刚刚说到的两种，对称加密跟非对称加密 数据是使用对称加密，对称加密过程需要客户端的一个密钥，为了确保能把该密钥安全传输到服务器端 ，采用非对称加密对该密钥进行加密传输，

简单来说：
1.对数据进行对称加密
2.对称加密锁需要的密钥通过非对称加密传输

HTTPS在传输过程

HTTPS在传输过程中会涉及到三个密钥：

1.服务器端的公钥跟密钥，用来进行非对称加密
2.客户端生成的随机密钥，用来进行对称加密

一个HTTPS请求实际上包含了两次HTTP传输，可以细分为8步

1.客户端想服务器发起HTTPS请求，链接到服务器的443端口

2.服务器端有一个密钥对，即公钥跟私钥，是用来进行非对称加密使用的， 服务器保存私钥，不能将其泄露，公钥可以发送给任何人

3.服务器将字的公钥发送给客户端

4。客户端收到服务器端的公钥之后，会紧公钥进行检查，验证其是否合法，如果发现 公钥有问题，那么https传输就无法继续，严格的说，这里应该是验证服务器发送来的数据证书的 合法性，如果公钥合格，那么客户端会生成一个随机值，这个随机值就是用来进行对称加密的密钥 即客户端密钥，然后使用服务器的公钥对客户端密钥进行非对称加密，这个客户端密钥就成了密文， 自此，HTTPS中的一次HTTP请求结束

5。客户端会发起HTTPS中的第二个http请求，将加密后的客户端密钥发送给服务器

6。服务器收到客户发来的密钥后，会使用自己的私钥对其进行对非对称解密， 解密后的明文就是客户端密钥，然后对客户端密钥进行对称加密，这样数据就称了密文

7。然后服务器将加密后的秘文发送给客户端

8。客户端接受到服务器发送过来的密文后，会使用客户密钥进行对称解密，得到 服务器发来的数据，这样https的第二个HTTP请求结束，这个http传输完成

HTTPS和HTTP的区别主要如下：

1、https协议需要到ca申请证书，一般免费证书较少，因而需要一定费用。

2、http是超文本传输协议，信息是明文传输，https则是具有安全性的ssl加密传输协议。

3、http和https使用的是完全不同的连接方式，用的端口也不一样，前者是80，后者是443。

4、http的连接很简单，是无状态的；HTTPS协议是由SSL+HTTP协议构建的可进行加密传输、 身份认证的网络协议，比http协议安全。

HTTPS的缺点

（1）HTTPS协议握手阶段比较费时，会使页面的加载时间延长近50%，增加10%到20%的耗电；

（2）HTTPS连接缓存不如HTTP高效，会增加数据开销和功耗，甚至已有的安全措施也会因此而受到影响；

（3）SSL证书需要钱，功能越强大的证书费用越高，个人网站、小网站没有必要一般不会用。

（4）SSL证书通常需要绑定IP，不能在同一IP上绑定多个域名，IPv4资源不可能支撑这个消耗。 　　

（5）HTTPS协议的加密范围也比较有限，在黑客攻击、拒绝服务攻击、服务器劫持等方面几乎起不到什么作用。最关键的，SSL证书的信用链体系并不安全，特别是在某些国家可以控制CA根证书的情况下，中间人攻击一样可行。
